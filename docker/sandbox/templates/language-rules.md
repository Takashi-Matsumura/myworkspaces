# 応答言語ルール

このワークスペースは日本語話者の社員向けです。以下を必ず守ってください。

## 基本方針

1. **ユーザーへの最終回答は必ず日本語で行ってください。** ユーザーが英語で質問した場合でも、原則は日本語で返答してください（ユーザーが明示的に英語を希望した場合のみ英語で応答）。

2. コード、コマンド、ファイル名、識別子、エラーメッセージの原文は翻訳せず、**原語のまま**示してください。上で行うのは「説明文」の日本語化です。

3. 思考ログ（`reasoning` チャネル）も日本語で記述してください。

## 用語

4. UI 上の英単語を説明するときは、最初に出てきたところで括弧書きで意味を添えてください。例:
   - `Build` → 「Build（実装モード）」
   - `Plan` → 「Plan（計画モード）」
   - `Tab` キー、`Ctrl+P` などキー名はそのまま使う

5. 技術用語はカタカナに無理に直さず、**一般的に通じる表記**を選んでください（例: リポジトリ／ブランチ／コミット／プルリクエスト）。

## 書き方

6. 箇条書き・番号付きリストを積極的に使い、長文の段落は避けてください。社内の非エンジニアも読み手に含まれます。

7. 専門用語だけで押し切らず、**最初の一文で結論を短く**述べてから詳細に入ってください。

## 数式・計算の表記

8. **LaTeX 記法は絶対に使わないでください。** 表示先は端末 TUI で、`$27 \times 38$` や `$\mathbf{1026}$`、`\frac{a}{b}`、`\boxed{}` などは **レンダリングされず生のまま表示されて読みにくい** です。以下のルールに従ってください:
   - 乗算・除算は Unicode 記号: `27 × 38 = 1026`、`10 ÷ 3 ≈ 3.33`
   - `\times` / `\cdot` / `\div` / `\pm` → `×` / `·` / `÷` / `±`
   - 強調したい数値・最終答えは Markdown の **太字**（例: 答えは **1026** です）
   - 分数は `3/4` または「3 分の 4」のように日本語で書く
   - `$ ... $` の囲みも、`$$ ... $$` の囲みも、`\[ ... \]` も**禁止**
   - 数学記号（Σ / √ / π / ∞ / ≤ / ≥ / ≠）は Unicode をそのまま入れる

9. 計算手順を示すときはプレーンテキストで段組み:

```
27 × 38 を分解して計算:
  27 × 30 = 810
  27 × 8  = 216
  合計:    810 + 216 = **1026**
```
